## Налоговое соответствие

Revroute Partners автоматически обеспечивает налоговое соответствие для ваших партнёров из США, собирая [формы W-9](https://support.stripe.com/express/questions/what-is-a-w-9-form) и отправляя [формы 1099-NEC](https://support.stripe.com/questions/intro-to-1099-nec-tax-forms-for-platforms-and-marketplaces) физическим лицам или организациям, получившим $600 и более в виде выплат за данный финансовый год.

Для партнёров за пределами США Revroute собирает [формы W-8](https://support.stripe.com/express/questions/what-is-a-w-8-form) для подтверждения того, что они не являются налоговыми резидентами США.

## Панель управления выплатами

Для просмотра выплат перейдите на вкладку **Partner Program** и нажмите на страницу **Payouts**.

![Вкладка выплат партнёрам](https://assets.dub.co/cms/partner-payouts-location.png)

В верхней части страницы вы увидите сводку по ожидающим и завершённым выплатам:

![Заголовок сводки по выплатам](https://assets.dub.co/cms/payout-summary-header.png)

**Ожидающие выплаты**

Здесь показана общая сумма комиссий, готовых к оплате.

При наведении на сумму отобразится:

- **Eligible payouts** — причитающиеся комиссии для партнёров, которые [настроили метод выплат](/help/partnering-with-programs/receiving-payouts/receiving-payouts)).
- **Ineligible payouts** — причитающиеся комиссии для партнёров, которые ещё [не настроили метод выплат](/help/partnering-with-programs/receiving-payouts/receiving-payouts)).

Когда у вас есть хотя бы одна ожидающая выплата в Revroute, вы можете нажать кнопку **Confirm payouts**, чтобы [проверить и подтвердить их](/help/managing-a-program/sending-payouts/partner-payouts#how-to-pay-your-partners).

**Всего выплачено**

Здесь показана общая сумма всех комиссий, выплаченных вашим партнёрам.

При наведении на сумму отобразится:

- **Completed payouts** — выплаты, которые были отправлены и подтверждены
- **Processing payouts** — выплаты, которые находятся в процессе (сроки могут варьироваться в зависимости от [выбранного метода выплат](/help/managing-a-program/sending-payouts/partner-payouts#payout-fees-and-timing))

Нажмите **View invoices**, чтобы получить доступ к прошлым счетам выплат.

В таблице выплат ниже вы можете отслеживать и управлять выплатами, подтверждать ожидающие и просматривать прошлые.

![Таблица выплат](https://assets.dub.co/cms/payouts-table.png)

| Столбец | Описание                                                                                                                         |
| ------- | -------------------------------------------------------------------------------------------------------------------------------- |
| Period  | Диапазон выплаты, начиная с даты первой комиссии партнёра до конца месяца последней комиссии                                     |
| Partner | Партнёр, получающий выплату                                                                                                      |
| Status  | Текущий [статус](/help/managing-a-program/sending-payouts/partner-payouts#payout-statuses) выплаты                                                          |
| Paid    | Дата подтверждения выплаты и кем                                                                                                 |
| Amount  | Общая сумма, выплаченная партнёру                                                                                                |

## Статусы выплат

Выплата может иметь один из следующих статусов:

| Статус     | Описание                                                                                                                                                                                                                                                                                              |
| ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Pending    | Выплаты, которые ещё не обработаны. Они могут быть текущими или оставаться в статусе pending (если партнёр не [настроил метод выплат](/help/partnering-with-programs/receiving-payouts/receiving-payouts), или выплата не достигла [минимальной суммы](#minimum-payout-amount)).                                                           |
| Processing | Выплаты, которые были [подтверждены](#how-to-pay-your-partners) — в зависимости от [метода выплат](#payout-fees-and-timing) этот этап может занять до 4 рабочих дней.                                                                                                                                 |
| Sent       | Средства отправлены на подключённый банковский счёт партнёра.                                                                                                                                                                                                                                         |
| Completed  | Выплата полностью завершена.                                                                                                                                                                                                                                                                          |
| Failed     | Это пограничный случай, возникающий при выплатах через PayPal, когда PayPal партнёра не настроен для получения выплат. С вашей стороны действий не требуется — партнёру нужно разобраться со своим аккаунтом PayPal и повторить выплату.                                                                |
| Canceled   | Для выплат, которые были отменены — обычно при блокировке партнёра в программе.                                                                                                                                                                                                                       |

## Период удержания выплат

В Revroute вы также можете установить «период удержания выплат», чтобы указать, как долго комиссии будут находиться в статусе «pending» до того, как станут доступны для выплаты.

Это полезно для защиты от потенциальных возвратов, чарджбеков или мошеннических транзакций.

<Info>
  Если вы используете нашу [интеграцию со Stripe](/docs/integrations/stripe),
  любые возвраты, произошедшие в Stripe, будут автоматически обнаружены Revroute, и
  [соответствующая комиссия также будет помечена как
  возвращённая](/help/managing-a-program/rewards-and-commissions/partner-commissions#commission-statuses).
</Info>

Чтобы установить период удержания для вашей программы, перейдите в [отдельные группы](/help/managing-a-program/getting-started/partner-groups) и нажмите на вкладку **Settings**. Здесь вы найдёте настройки периода удержания в группе «Additional settings».

![Настройки выплат программы](https://assets.dub.co/cms/group-additional-settings.png)

Доступные варианты:

- 0 дней (по умолчанию)
- 7 дней
- 14 дней
- 30 дней
- 60 дней
- 90 дней

При обновлении периода удержания вы можете применить новую настройку ко всем группам сразу. Просто установите флажок `Apply to all groups` перед подтверждением. Изменение вступит в силу немедленно.

![Период удержания выплат](https://assets.dub.co/cms/confirm-holding-period.png)

## Минимальная сумма выплаты

Вы также можете установить необязательную «минимальную сумму выплаты» (например, $100). При установке партнёры с заработком ниже минимальной суммы не будут доступны для выплаты до достижения указанного порога.

Нажмите кнопку в правом верхнем углу, чтобы отобразить **Payout settings**. В отличие от периода удержания, эта настройка действует для всей программы.

![Кнопка настроек выплат](https://assets.dub.co/cms/program-payout-settings.png)

Доступные варианты:

- $0 (по умолчанию)
- $10
- $20
- $50
- $100

![Настройки выплат](https://assets.dub.co/cms/payout-settings-drawer.png)

## Как платить партнёрам

Когда один или несколько ваших партнёров заработали больше минимальной суммы выплаты, нажмите **Confirm payouts** в [заголовке сводки](/help/managing-a-program/sending-payouts/partner-payouts#summary-header), чтобы открыть детали выплаты.

- **Payout method:** Выберите метод выплаты на основе подключённых аккаунтов. Нажмите значок шестерёнки для управления [подключёнными методами оплаты](https://app.dub.co/settings/billing).
- **Cutoff Period:** Если установлен, будут оплачены только комиссии, заработанные до этой даты включительно. Полезно для сверочных выплат за определённый финансовый период (например, прошлый месяц, квартал) без включения более новых комиссий.

Если всё верно, нажмите **Confirm payout**, чтобы инициировать выплату партнёрам.

### Исключение выплат

В некоторых случаях вы можете захотеть исключить выплату партнёра из текущей партии. Для этого наведите курсор на строку партнёра в таблице выплат и нажмите **Exclude**. Это удалит партнёра из текущей партии выплат.

![Исключение выплат](/images/dub-partners/exclude-payout.png)

Если нужно добавить партнёра обратно, наведите курсор на его строку и нажмите **Include**. Это вернёт его в текущую партию выплат.

## Комиссии и сроки выплат

В Revroute вы можете отправлять выплаты различными методами оплаты, такими как ACH, кредитная карта и другие. В зависимости от вашего тарифа Revroute и выбранного метода оплаты комиссии и сроки будут различаться:

| Метод оплаты                                                         | Комиссия                                                                                             | Сроки                 |
| -------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- | --------------------- |
| [ACH](/help/managing-a-program/sending-payouts/how-to-set-up-bank-account)                      | [Business / Advanced](https://dub.co/pricing): 5% <br/> [Enterprise](https://dub.co/enterprise): 3%  | До 4 рабочих дней     |
| Credit card / [Link](https://link.com/) <br/> Apple Pay / Google Pay | [Business / Advanced](https://dub.co/pricing): 8% <br/> [Enterprise](https://dub.co/enterprise): 6%  | Мгновенно             |

Для крупных выплат мы рекомендуем использовать ACH как наиболее экономичный вариант. Вот [как настроить банковский счёт](/help/managing-a-program/sending-payouts/how-to-set-up-bank-account) для выплат через ACH.
