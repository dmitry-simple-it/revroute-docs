# API: ошибки

Ошибки RevRoute API возвращаются в едином формате:

```json
{
  "error": {
    "code": "unprocessable_entity",
    "message": "Invalid input",
    "doc_url": "https://revroute.ru/docs/api-reference/errors#unprocessable-entity"
  }
}
```

## `unprocessable_entity` (422)

Запрос синтаксически корректный, но не проходит валидацию (например, неверный тип поля).

Частый пример:

- `testVariants` ожидается **массив**, а пришёл **объект**

Проверьте тело запроса и приведите его к ожидаемой схеме.

## `unauthorized` (401)

Нет токена или токен недействителен.

## `forbidden` (403)

Токен валиден, но не хватает прав доступа.

## `not_found` (404)

Ресурс не найден или вам недоступен.
