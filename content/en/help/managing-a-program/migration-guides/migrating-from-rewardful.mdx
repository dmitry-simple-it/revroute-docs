Rewardful is a popular affiliate management platform, especially for businesses that use Stripe for payments.

However, Rewardful can be pretty limiting for companies that want to run their affiliate programs at scale:

1. **No payouts support**: Companies generally have to resort to using PayPal/Wise to pay out their affiliates manually and deal with the all-too-common payout failures.
2. **Basic reward structures:** Companies can choose between two reward types: recurring or one-time sales rewards.
3. **Limited white-labeling support**: It's not possible to embed Rewardful's dashboard inside your application.
4. **No affiliate network**: Rewardful is a standalone affiliate software, which means you're on your own when it comes to finding and recruiting high-quality affiliates to join your program.

**This is where [Revroute Partners](https://revroute.ru/) comes in.**

1. **Payouts you can rely on:** We let you [pay your partners globally](/help/managing-a-program/sending-payouts/partner-payouts) in 1-click, with tax compliance built-in, so you can have peace of mind knowing that all payouts are completed successfully.
2. **Advanced reward structures**: On Revroute, you can choose between [3 reward types](/help/managing-a-program/rewards-and-commissions/partner-rewards#configuring-reward-types) – per click, per lead, and per sale (flat/percentage). You can also set different [reward conditions](/help/managing-a-program/rewards-and-commissions/partner-rewards#adding-reward-conditions) (e.g., a higher reward for customers based in the U.S. vs other countries) and create [bounties](/help/managing-a-program/engaging-partners/program-bounties) to provide additional rewards for your affiliates.
3. **White-labeling**: You can [embed a white-labeled referral dashboard into your application](/docs/partners/embedded-referrals) for a better user experience.
4. **Powerful affiliate network**: With over 5,000 active affiliates in our network, Revroute makes it easy to find and recruit high-performing partners with a proven track record of driving results.

## How to migrate from Rewardful to Revroute?

If you're looking to migrate from Rewardful to Revroute Partners, we've built a custom migration tool that will help you migrate everything over in just 2-3 clicks – without breaking any existing Rewardful affiliate links.

<Steps>
<Step title="Add a custom domain to your Revroute workspace">
Before starting the migration process, you'll need to add a custom domain to Revroute. This domain will be used to create short links for each affiliate link that we import from Rewardful.

If you haven't added a custom domain to Revroute yet, you can [follow this guide to set that up](/help/managing-your-links/custom-domains/how-to-add-custom-domain).

</Step>

<Step title="Set up conversion tracking + client-side click-tracking">

Next, [set up conversion tracking](/docs/quickstart/server) to track signup and sale events. We also have some [in-app guides that you can reference here](https://app.revroute.ru/guides).

For backward compatibility, we also recommend setting up [client-side click-tracking](/docs/sdks/client-side/features/click-tracking) when installing the [Revroute Analytics script](/docs/sdks/client-side/introduction) on your site. This will ensure all your existing `domain.com?via=affiliate` links will keep working as expected, which gives affiliates time to update their links to the new format.

You can [follow our guide](/docs/sdks/client-side/features/click-tracking) on how to set up client-side click-tracking for your Revroute short links to set that up.

</Step>

<Step title="Get your Rewardful API Secret">
Go to your [Company settings page on Rewardful](https://app.getrewardful.com/company/edit) to retrieve your Rewardful API secret:

![Rewardful API secret](https://assets.dub.co/cms/rewardful-api-secret.png)

We will use this API key to retrieve and import your existing Rewardful campaign to Revroute.

</Step>

<Step title="Import your Rewardful campaign in the Partners tab">

Once you've [set up your Revroute partner program](/help/managing-a-program/getting-started/setting-up-your-program), go to your [Partners tab](https://app.revroute.ru/program/partners), click on the `⋮` button and select "Import from Rewardful".

To save you some time, [here's a quicklink](https://app.revroute.ru/program/partners?import=rewardful) that will get you directly to the Rewardful import flow.

![Import Rewardful modal](https://assets.dub.co/cms/import-rewardful-modal.png)

Then, add your Rewardful API secret and select **Fetch campaigns**. From the retrieved list of Rewardful campaigns, select the main campaign you want to import first and click **Continue**.

<Tip>
  If you have multiple campaigns on Rewardful, you can [import multiple
  campaigns](#migrating-multiple-campaigns) into Revroute as well.
</Tip>

</Step>
</Steps>

## What data is migrated?

Our Rewarful migration tool imports the following attributes to Revroute Partners:

### Program terms

Depending on the Rewardful campaign you import, we'll automatically configure the same reward rules on your Revroute partner program:

- Commission type (dollar amount or percentage) and amount
- Commission duration (e.g. up to 12 months or lifetime recurring)

### Affiliates / Partners

To ensure the efficiency of your partner program, we automatically filter out dormant affiliates and only import the ones that are both:

- In an "active" state ([docs](https://developers.rewardful.com/rest-api/affiliates/object#state))
- Have at least 1 lead / referral

We also automatically import all your affiliates' links and recreate them in Revroute using your program's domain and default URL.

### Referrals / Customers

By default, we import all referrals that are associated with an affiliate in your Rewardful campaign. This ensures that all subsequent subscription renewal events will continue to be tracked on Revroute.

<Warning>
  Referrals that do not have a valid Stripe customer ID will not be imported.
</Warning>

### Commissions

We also import all commissions associated with any customers that were imported above. Depending on the commission state on Rewardful, they will be marked accordingly on Revroute as well:
| Rewardful | Revroute |
| --- | --- |
| `Pending` | `Pending` |
| `Due` | `Pending` |
| `Paid` | `Paid` |
| `Voided` | `Canceled` |

Additionally, Revroute also supports 3 additional commission states:

- `Processed`: When a commission has been added to a [payout](/help/managing-a-program/sending-payouts/partner-payouts) (after any applicable holding period).
- `Refunded`: When the sale for the commission was refunded. With our [Stripe integration](/docs/integrations/stripe), this is automatically detected and updated.
- `Fraud`: Useful for marking a sale as fraudulent and excluding it from any future payouts.
- `Duplicate`: Useful for marketing if a commission was rewarded more than once and needs to be marked as a duplicate event.

<Warning>
  Similar to referrals, commissions for a referral that do not have a valid
  Stripe customer ID will not be imported.
</Warning>

If there are any other attributes that you'd like to see migrated (or if you would like to customize any of the rules above), please [send us an email](mailto:support@revroute.ru?subject=Rewardful%20migration%20feedback) to let us know!

## Migrating multiple campaigns

If needed, you can migrate multiple Rewardful campaigns to Revroute. We recommend starting with the main campaign, and then restarting the import flow to import the remaining campaigns. Our importer is built in an idempotent fashion, so it will import everything only once.

To start another import flow, go to your program's **Partners** tab and click on the `⋮` button next to **Invite partner**. From the dropdown, select **Import from Rewardful**.

![Import from Rewardful](https://assets.dub.co/cms/import-from-rewardful.png)

This will open up the Rewardful importer, which you can then use to import more campaigns to Revroute.

![Import Rewardful modal](https://assets.dub.co/cms/import-rewardful-modal.png)

Each campaign that is migrated to Revroute will create a new [partner group](/help/managing-a-program/getting-started/partner-groups) with the imported partners. If needed, you can also delete the imported partner groups to consolidate partners under your main program reward.
