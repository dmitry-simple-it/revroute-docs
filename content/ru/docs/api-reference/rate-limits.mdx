# API: лимиты запросов

RevRoute API применяет rate limiting, чтобы обеспечить стабильность сервиса и защиту от злоупотреблений.

## Как понять, что вы упёрлись в лимит

Если лимит превышен, вы получите:

- HTTP `429 Too Many Requests`

и заголовки, помогающие корректно повторить запрос позже:

- `X-RateLimit-Limit` — сколько запросов разрешено в окне лимита
- `X-RateLimit-Remaining` — сколько осталось
- `X-RateLimit-Reset` — когда окно сбросится (epoch seconds, UTC)
- `Retry-After` — через сколько секунд можно повторить

## Как соблюдать лимиты

- Используйте бэкенд‑интеграции и кеширование вместо частого опроса из браузера.
- По возможности отправляйте события пачкой (если ваш сценарий это позволяет).
- Для real‑time сценариев используйте вебхуки вместо polling.
