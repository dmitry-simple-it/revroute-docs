## How to migrate from PartnerStack to Revroute?

If you're looking to migrate from PartnerStack to Revroute Partners, we've built a custom migration tool that will help you migrate everything over in just 2-3 clicks – without breaking any existing PartnerStack affiliate links.

<Steps>
<Step title="Add a custom domain to your Revroute workspace">
Before starting the migration process, you'll need to add a custom domain to Revroute. This domain will be used to create short links for each affiliate link that we import from PartnerStack.

If you haven't added a custom domain to Revroute yet, you can [follow this guide to set that up](/help/managing-your-links/custom-domains/how-to-add-custom-domain).

</Step>

<Step title="Set up conversion tracking">

Next, [set up conversion tracking](/docs/quickstart/server) to track signup and sale events. We also have some [in-app guides that you can reference here](https://app.revroute.ru/guides).

</Step>

<Step title="Get your PartnerStack API Keys">

[Follow this guide](https://docs.tolt.com/authentication) to get your PartnerStack API keys – API keys can be created in the [Settings → API](https://app.partnerstack.com/settings/integrations) tab of your PartnerStack dashboard:

![PartnerStack API key](https://assets.dub.co/cms/partnerstack-api-key.png)

<Tip>
  Make sure to use your production API keys to import live partners only (since
  test partners won't be imported).
</Tip>

We will use the generated Public and Secret keys to retrieve and import your existing PartnerStack program to Revroute.

</Step>

<Step title="Import your PartnerStack program in the Partners tab">

Once you've [set up your Revroute partner program](/help/managing-a-program/getting-started/setting-up-your-program), go to your [Partners tab](https://app.revroute.ru/program/partners), click on the `⋮` button and select "Import from PartnerStack".

To save you some time, [here's a quicklink](https://app.revroute.ru/program/partners?import=partnerstack) that will get you directly to the PartnerStack import flow.

![Import PartnerStack modal](https://assets.dub.co/cms/import-partnerstack-modal.png)

Then, add your PartnerStack Public and Secret keys and select **Continue.**

</Step>

</Steps>

## What data is migrated?

Our PartnerStack migration tool imports the following attributes to Revroute Partners:

### Affiliates / Partners

To ensure the efficiency of your partner program, we automatically filter out dormant affiliates and only import the ones that are both:

- In an "active" state ([docs](https://developers.rewardful.com/rest-api/affiliates/object#state))
- Have at least 1 lead / referral

We also automatically import all your affiliates' links and recreate them in Revroute using your program's domain and default URL.

<Info>
Since PartnerStack doesn't expose a `/triggers` endpoint, your PartnerStack triggers will not be migrated – all partners will be assigned to your program's default reward.

Learn more about how [Partner rewards](/help/managing-a-program/rewards-and-commissions/partner-rewards) work on Revroute.

Feel free to [reach out to support](https://revroute.ru/contact/support) if you need help manually configuring specific reward groups for your importer partners.

</Info>

### Referrals / Customers

By default, we import all referrals that are associated with an affiliate in your PartnerStack campaign. This ensures that all subsequent subscription renewal events will continue to be tracked on Revroute.

Certain customers will not be imported, such as test accounts or any customer that is not associated with an affiliate link.

<Warning>
  Some referrals might not have a valid Stripe customer ID. Those referrals will
  still be imported, with the Stripe customer ID updated afterwards.
</Warning>

### Commissions

We also import all commissions associated with any customers that were imported above. Depending on the commission state on PartnerStack, they will be marked accordingly on Revroute as well:
| PartnerStack | Revroute |
| --- | --- |
| `Pending` | `Pending` |
| `Approved` | `Processed` |
| `Paid` | `Paid` |
| `Declined` | `Canceled` |
| `Hold` | `Fraud` |

Additionally, Revroute also supports 2 additional commission states:

- `Refunded`: When the sale for the commission was refunded. With our [Stripe integration](/docs/integrations/stripe), this is automatically detected and updated.
- `Duplicate`: Useful for marketing if a commission was rewarded more than once and needs to be marked as a duplicate event.

<Warning>
  Commissions for a referral that do not have a valid Stripe customer ID or a
  transaction trace will not be imported.
</Warning>

If there are any other attributes that you'd like to see migrated (or if you would like to customize any of the rules above), please [send us an email](mailto:support@revroute.ru?subject=Rewardful%20migration%20feedback) to let us know!
