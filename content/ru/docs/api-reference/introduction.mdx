# API: введение

RevRoute API позволяет программно управлять ресурсами вашего воркспейса: ссылками, доменами, событиями и аналитикой.

## Базовый URL

Все запросы выполняются по HTTPS.

- Base URL: `https://api.revroute.ru`

## Аутентификация

API использует заголовок `Authorization` с токеном формата Bearer:

```http
Authorization: Bearer <ваш_токен>
```

Токены создаются и управляются в кабинете RevRoute в разделе **Tokens** (Настройки → Токены). Никогда не размещайте секретный токен в клиентском коде.

## Формат ошибок

При ошибках API возвращает машинно‑читаемый `code`, человеко‑читаемый `message` и ссылку на документацию.

```json
{
  "error": {
    "code": "unprocessable_entity",
    "message": "Invalid input",
    "doc_url": "https://revroute.ru/docs/api-reference/errors#unprocessable-entity"
  }
}
```

## Пагинация

Листинги обычно поддерживают параметры:

- `page` — номер страницы
- `pageSize` — размер страницы
- `sortBy` / `sortOrder` — сортировка

Конкретные параметры могут отличаться для разных эндпоинтов.
