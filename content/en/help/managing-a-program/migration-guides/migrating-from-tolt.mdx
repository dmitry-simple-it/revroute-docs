Tolt is a go-to affiliate management platform for many early-stage companies.

However, Tolt is limiting for companies that want to run their affiliate programs at scale:

1. **Basic analytics**: While Tolt shows basic clicks and leads, it lacks UTM and referrer data. You see the numbers but not what drives them.
2. **Frequent payout failures**: Tolt offers automated payouts on higher-tier plans, but customers who migrated to Revroute reported frequent payout failures. This means spending hours troubleshooting payments and managing frustrated partners.
3. **No affiliate network**: As standalone software, Tolt leaves the task of finding and recruiting high-quality affiliates entirely to you.
4. **Outdated UI:** Tolt’s UI feels outdated, despite the platform positioning itself for SaaS companies, where intuitive, modern dashboards are typically a priority.

**This is where [Revroute Partners](https://revroute.ru/) comes in.**

1. **Comprehensive analytics**: Revroute offers [powerful real-time analytics](https://revroute.ru/), including [conversion tracking](/help/article/dub-conversions), device, browser, referrer, country, and more. Our [AI feature](/help/managing-your-links/analytics/dub-analytics#bonus-ask-ai-feature) makes filtering data intuitive and quick.
2. **Payouts you can rely on:** We let you [pay your partners globally](/help/managing-a-program/sending-payouts/partner-payouts) in 1-click, with tax compliance built-in, so you can have peace of mind knowing that all payouts are completed successfully.
3. **Powerful affiliate network**: With over 5,000 active affiliates in our network, Revroute makes it easy to find and recruit high-performing partners with a proven track record of driving results.
4. **Advanced reward structures**: On Revroute, you can choose between [3 reward types](/help/managing-a-program/rewards-and-commissions/partner-rewards#configuring-reward-types) – per click, per lead, and per sale (flat/percentage). You can also set different [reward conditions](/help/managing-a-program/rewards-and-commissions/partner-rewards#adding-reward-conditions) (e.g., a higher reward for customers based in the U.S. vs other countries) and create [bounties](/help/managing-a-program/engaging-partners/program-bounties) to provide additional rewards for your affiliates.

In this article, we'll learn how to import your Tolt affiliate program to[ Revroute Partners](/help/managing-a-program/dub-partners) for a more powerful, scalable, and branded partner management experience.

## How to migrate from Tolt to Revroute?

If you're looking to migrate from Tolt to Revroute Partners, we've built a custom migration tool that will help you migrate everything over in just 2-3 clicks – without breaking any existing Tolt affiliate links.

<Steps>
<Step title="Add a custom domain to your Revroute workspace">
Before starting the migration process, you'll need to add a custom domain to Revroute. This domain will be used to create short links for each affiliate link that we import from Tolt.

If you haven't added a custom domain to Revroute yet, you can [follow this guide to set that up](/help/managing-your-links/custom-domains/how-to-add-custom-domain).

</Step>

<Step title="Set up conversion tracking + client-side click-tracking">

Next, [set up conversion tracking](/docs/quickstart/server) to track signup and sale events. We also have some [in-app guides that you can reference here](https://app.revroute.ru/guides).

For backward compatibility, we also recommend setting up [client-side click-tracking](/docs/sdks/client-side/features/click-tracking) when installing the [Revroute Analytics script](/docs/sdks/client-side/introduction) on your site. This will ensure all your existing `domain.com?via=affiliate` links will keep working as expected, which gives affiliates time to update their links to the new format.

You can [follow our guide](/docs/sdks/client-side/features/click-tracking) on how to set up client-side click-tracking for your Revroute short links to set that up.

</Step>

<Step title="Get your Tolt API Key and Program ID">
[Follow this guide](https://docs.tolt.com/authentication) to get your Tolt API key – API keys can be created in the [Settings →  Integrations](https://app.tolt.io/settings?tab=integrations) tab of your Tolt dashboard:

![Tolt API key](https://assets.dub.co/cms/tolt-api-key.png)

You will also need to get your Tolt Program ID under [Program Settings](https://app.tolt.io/program-settings?page=general-settings):

![Tolt Program ID](https://assets.dub.co/cms/tolt-program-id.png)

We will use the API key and Program ID to retrieve and import your existing Tolt affiliate program to Revroute.

</Step>

<Step title="Import your Tolt program in the Partners tab">

Once you've [set up your Revroute partner program](/help/managing-a-program/getting-started/setting-up-your-program), go to your [Partners tab](https://app.revroute.ru/program/partners), click on the `⋮` button and select "Import from Tolt".

To save you some time, [here's a quicklink](https://app.revroute.ru/program/partners?import=tolt) that will get you directly to the Tolt import flow.

![Import Tolt modal](https://assets.dub.co/cms/import-tolt-modal.png)

Then, add your Tolt API key + Program ID and select **Continue.**

</Step>

</Steps>

## What data is migrated?

Our Tolt migration tool imports the following attributes to Revroute Partners:

### Affiliates / Partners

To ensure the efficiency of your partner program, we automatically filter out dormant affiliates and only import the ones that are both:

- In an "active" state ([docs](https://developers.rewardful.com/rest-api/affiliates/object#state))
- Have at least 1 lead / referral

We also automatically import all your affiliates' links and recreate them in Revroute using your program's domain and default URL.

<Info>
Since Tolt doesn't expose a `/rewards` endpoint, your Tolt program terms will not be migrated – all partners will be assigned to your program's default reward.

Learn more about how [Partner rewards](/help/managing-a-program/rewards-and-commissions/partner-rewards) work on Revroute.

</Info>

### Referrals / Customers

By default, we import all referrals that are associated with an affiliate in your Tolt program. This ensures that all subsequent subscription renewal events will continue to be tracked on Revroute.

<Warning>
  Some referrals might not have a valid Stripe customer ID. Those referrals will
  still be imported, with the Stripe customer ID updated afterwards.
</Warning>

### Commissions

We also import all commissions associated with any customers that were imported above. Depending on the commission state on Tolt, they will be marked accordingly on Revroute as well:
| Tolt | Revroute |
| --- | --- |
| `Pending` | `Pending` |
| `Approved` | `Pending` |
| `Paid` | `Paid` |
| `Rejected` | `Canceled` |
| `Refunded` | `Refunded` |

Additionally, Revroute also supports 3 additional commission states:

- `Fraud`: Useful for marking a sale as fraudulent and excluding it from any future payouts.
- `Processed`: When a commission has been added to a [payout](/help/managing-a-program/sending-payouts/partner-payouts) (after any applicable holding period).
- `Duplicate`: Useful for marketing if a commission was rewarded more than once and needs to be marked as a duplicate event.

<Warning>
  Commissions for a referral that do not have a valid Stripe customer ID or a
  transaction trace will not be imported.
</Warning>

If there are any other attributes that you'd like to see migrated (or if you would like to customize any of the rules above), please [send us an email](mailto:support@revroute.ru?subject=Rewardful%20migration%20feedback) to let us know!
