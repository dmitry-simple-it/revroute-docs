## Где найти настройки реферальных ссылок

Каждая [группа партнеров](/help/managing-a-program/getting-started/partner-groups) имеет собственные настройки реферальных ссылок. Чтобы найти их, откройте группу и нажмите **Links** в верхней навигации.

![Настройки реферальных ссылок группы](https://assets.dub.co/cms/group-link-settings-main.png)

## Реферальные ссылки по умолчанию

Ссылка по умолчанию создается при настройке программы, поэтому новые [группы партнеров](/help/managing-a-program/getting-started/partner-groups) всегда имеют реферальную ссылку. Вы также можете настроить дополнительные ссылки по умолчанию, которые автоматически создаются при добавлении партнера в группу.

Нажмите **Create Link**, чтобы добавить дополнительные ссылки по умолчанию:

![Ссылки по умолчанию группы](https://assets.dub.co/cms/default-links.png)

<Info>
  При создании ссылки по умолчанию вас попросят указать **Destination URL**.
  Это страница, на которую попадают пользователи после нажатия на реферальную
  ссылку. Обычно это ваша главная страница или специальная целевая страница
  программы.
</Info>

**Поведение ссылок по умолчанию**

- **При создании ссылки по умолчанию** новая ссылка генерируется для каждого партнера в этой группе. Это относится как к текущим, так и к будущим партнерам в этой группе.
- **При обновлении URL назначения ссылки по умолчанию** обновляются все соответствующие партнерские ссылки в этой группе с указанным URL назначения.
- **При удалении ссылки по умолчанию** ссылка остается у существующих партнеров в этой группе для сохранения исторической статистики/комиссий.

## Дополнительные партнерские ссылки

Вы также можете позволить вашим партнерам создавать собственные ссылки помимо стандартных:

![Дополнительные партнерские ссылки](https://assets.dub.co/cms/additional-partner-links.png)

Для дополнительных партнерских ссылок вы можете указать следующее:

- **Link limit**: максимальное количество ссылок, которые могут создать партнеры
- **Link domains**: домены, которым должны соответствовать ссылки ваших партнеров (любая страница на домене или только главная страница)

![Добавление домена ссылки](https://assets.dub.co/cms/adding-link-domain.png)

<Warning>
  Убедитесь, что на добавляемом домене настроено отслеживание конверсий,
  иначе ваши ссылки не будут отслеживаться корректно. [Подробнее.](/docs/quickstart/server)
</Warning>

## Настройки ссылок

Определите структуру ссылок и UTM-параметры для всех партнеров в группе.

![Настройки ссылок](https://assets.dub.co/cms/link-settings.png)

### Структура ссылок

Revroute поддерживает два формата реферальных ссылок. Выберите в зависимости от того, как вы хотите, чтобы ссылки выглядели и где они будут использоваться. Оба формата работают одинаково для атрибуции.

![Структура ссылок](https://assets.dub.co/cms/link-structure.png)

<Info>
  Вы можете использовать любые типы реферальных ссылок взаимозаменяемо — опция
  выше определяет, какой формат отображается в панели партнера.
</Info>

**Вариант 1: Короткие ссылки**

**Формат:** `getacme.link/{partnerName}`

Чистая, брендированная короткая ссылка, которую легко запомнить и удобно использовать в публикациях в социальных сетях, биографиях или сообщениях.

<Tip>
  Мы рекомендуем использовать короткие ссылки по нескольким причинам:

  1. Некоторые браузеры блокируют [отслеживание кликов на стороне клиента](/docs/sdks/client-side/features/click-tracking), что нарушает атрибуцию — с `getacme.link/john` клик будет отслеживаться и на стороне сервера, что повышает точность атрибуции
  2. Некоторые браузеры удаляют параметры запроса, что также нарушает атрибуцию
  3. Это выглядит аккуратнее
</Tip>

**Вариант 2: Параметр запроса**

**Формат:** `acme.com?via={partnerName}`

Добавляет имя партнера в качестве строки запроса к вашему URL назначения. Идеально для пользователей, которые [мигрируют с Rewardful](/help/managing-a-program/migration-guides/migrating-from-rewardful) и хотят сохранить обратную совместимость существующих партнерских ссылок.

### UTM-параметры

Установите [UTM-параметры](/help/managing-your-links/links/utm-builder) по умолчанию для всех ссылок в этой группе. Это дает вам аналитику по конкретным группам.

![UTM ссылок](https://assets.dub.co/cms/link-utm.png)

При обновлении [шаблона UTM](/help/managing-your-links/links/how-to-create-utm-templates) группы все партнерские ссылки в
группе обновляются и перезаписываются новыми UTM-тегами.

## Как изменение группы партнеров влияет на реферальные ссылки

Как упоминалось ранее, настройки ссылок партнера привязаны к [группе партнеров](/help/managing-a-program/getting-started/partner-groups), в которой он состоит. При обновлении группы партнера или удалении групп его партнерские ссылки также будут обновлены.

### Перемещение партнеров в группу

При перемещении партнера в новую группу его ссылки автоматически обновляются в соответствии с настройками по умолчанию и UTM-параметрами новой группы.

<Info>
  **Зачем перемещать партнеров?**

  - Для повышения их с начальной группы в группу с более высоким уровнем вознаграждения
  - Для тестирования новых вознаграждений в тестовой группе
  - Для упорядочивания структуры и общей реорганизации
</Info>

### Удаление групп

При удалении группы все партнеры в этой группе перемещаются в группу по умолчанию. Их ссылки обновляются в соответствии с настройками группы по умолчанию.

<Info>
  **Зачем удалять группу?**

  - Кампания завершена
  - Вы создали дублирующую группу по ошибке
  - Бета-тестовая группа больше не нужна
</Info>
